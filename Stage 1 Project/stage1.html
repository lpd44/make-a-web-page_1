Lesson 1: The Basics of the Web and HTML

How the Web Works

The web is a series of discrete systems or components that work together. They include:

User + Browser

A user on a computer or similar device, uses a browser, or similar application to make requests to servers. Browsers include Chrome, Firefox, etc.

Internet

The internet is the communication platform that transmits requests from users to servers. When requesting a web page a browser uses HTTP (Hypertext Transfer Protocol).

Servers

Servers are specialised computers, optimized to deliver content based on user requests.

How it all works

When the user, in their browser tries to open a webpage, a HTTP request is sent from the computer, over the internet, to the correct server that is storing that web page. The server find the correct HTML document for the requested web page and returns it back, over HTTP to the users computer, where it is rendered by the browser.

HTML

HTML stands for Hypertext Markup Language.

Most of the content on the web is made up from HTML. HTML documents contain:

Text content: The text you see (this text for example:)). Remember all white space without markup turns into 1 single space!

Markup: The formating or, how things are displayed.

References to other Docs: Other HTML Docs, etc.

Links: Nuff' said.

Tags and Elements

Tags implement markup in HTML.
An element is a component of a HTML Doc, usually having a opening and a closing tag.
A opening tag is like <p>.
A closing tag is like </p>.
A note in the img tag; the alt attribute is not required to be populted, but can be useful for SEO, blind people, etc. This is an example of a VOID element - it has no closing tag.

If the closing tag is omitted (but required), the change implemented by the opening tag is continued to the end of the document/until another matching close is found. This can have unexpected effects.

Inline vs Block Elements

HTML elements can be either inline or block.
Block elements have a container, or an "invisible box". Inline are only in the line the tags are in.
<br> is an inline tag, along with span. <p> is block, along with div.

Things to be aware of when coding

Computers are stupid: They will try to interpret whatever is given to them, exactly as it is presented. So a typo can have dramatic effects.

Don't try to remember everything: Concentrate on the concepts. A Google can always be used to find the syntax

Learn the vocab: So you know what people are talking about

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS is the applied style for HTML.

A style attribute is applied to a HTML element. For example,

h1 {
background-color : black;
color: white;
}

The Importance of not repeating oneself

When complex applications begin to be created, it is important to ensure code is logical, easy to follow and understand and as efficient as possible.

A key element to this is re-use of code. If the same thing happens more than once, the code to implement the change should exist only once.

So if my body content, should all be formatted red, create the red formating rule once, then apply it to all body code.

Some of the benefits include:
1. Saving time by writing less code. Elements that share teh same style are all referenced from 1 css file.
2. Editing becomes much easier
3. The liklehood for errors is reduced. But reducing the amount of code and only have to declare one style once, the liklehood of errors is reduced. And you should quickly identifiy it as it will effect all elements that inherit that style.
